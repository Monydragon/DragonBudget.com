Launch plan

Use this checklist to move Dragon Budget from local site files to a public GitHub Pages site with crawlable content, a working custom domain, and a feedback loop.

GitHub Pages deployment

  1. Commit the root site files and the `docs` source documents.
  2. In GitHub Pages settings, publish from the main branch repository root.
  3. Keep the root `CNAME` file set to `DragonBudget.com`.
  4. Configure DNS for the custom domain and enable HTTPS when GitHub allows it.
  5. Confirm `https://dragonbudget.com/robots.txt` and `https://dragonbudget.com/sitemap.xml` load.

Root publish reminder

The site no longer publishes from `/docs`. The `docs` folder now contains internal planning files and release tracking.
