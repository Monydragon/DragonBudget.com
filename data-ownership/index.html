Product direction

Data ownership philosophy

Dragon Budget should make user data portable, understandable, and recoverable through local-first storage goals, export plans, and clear backup workflows.

Manual-first privacy

Users should be able to start without bank sync or cloud accounts.

Export plans

CSV and JSON export are planned so users can leave with their data intact.

Backup plans

Local backup and import should include clear warnings before overwriting data.

Future sync boundary

Cloud sync or bank sync should be optional, documented, and separated behind architecture boundaries. The public site does not sync, store, or transmit financial inputs.
